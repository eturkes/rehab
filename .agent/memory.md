## 0b. Lessons & mistakes (append; prune when superseded)

* **Validate stats against a reference before committing.** Holm step-down needs a running **max** over sorted p×(n−k+1); a flipped `min` gave nonsensical ~10⁻⁵⁵ adjusted-p. Operator direction flips easily.
* **Plotly fails silently — check the data contract first.** `Sunburst(branchvalues="total")` with zero-valued parents renders blank, no error/log. Empty chart ⇒ suspect the parent/child value contract (or `branchvalues`) before a data bug. See §5.
* **Heatmap: encode likelihood, not set-membership; force the category axis.** A magnitude APS heatmap keyed on binary z=APS-membership degenerated (set near-always full {0,+1,≥+2}); encode class *proba* with dynamic `zmax`. Heatmap with string x + few rows silently falls back to a numeric axis — set `xaxis/yaxis type="category"`. Keep the uncalibrated magnitude panel visually distinct (slate `PALETTE_CATEGORICAL[3]`, not teal) from the calibrated descent panel.
* **Verify a referenced symbol exists (grep first).** `_apply_missing_sentinels` was cited but never written; sentinels are produced by two other mechanisms (§1).
* **Test UI at minimum viewport width.** Full Japanese demographic strings wrapped and blended Dropdown options; `optionHeight` must match the rendered row height.
* **Investigate NaN / row-count anomalies immediately.** 301 ghost episodes inflated counts for ~5 sessions though visible from session 1 (filter now in loader, §1).
* **uv resolves unconstrained transitive deps to their MIN-compatible version.** shap's unconstrained `numba` pulled 2021 numba/llvmlite (no Py3.13). Add explicit lower bounds in `pyproject.toml`, then verify resolved versions in `uv.lock` after any upgrade.
* **`shap_interaction_values()` rejects category-dtype columns** (stricter than `shap_values()`). Encode categoricals to int codes (`cat.codes.astype(float)`) first; `shap_values()` handles LightGBM categoricals internally.
* **Relocating the project root breaks `.venv` wholesale (incl. `uv run`).** Stale absolute paths live in `.venv/bin` shebangs, `activate`'s `VIRTUAL_ENV=`, and the editable `rehab_sci` install. Fix: `rm -rf .venv && uv sync` (~1 min, offline via warm cache). The repo tree and `uv.lock` hold no absolute paths.
* **`grep` here is `ugrep`, not GNU.** BRE `\|` matches literally — use ERE `grep -E`. Re-check surprising empty sweeps.
* **Split a megafile by carving line-ranges + an AST-equivalence assert, not by retyping.** Slice each top-level symbol by its def-to-def range; assert every `FunctionDef`/const survives byte-identical (`ast.dump`) before deleting the monolith. Give each submodule the full import header; `ruff check --fix` prunes unused (F401); `ruff check --select F` (F401+F821) gates that no symbol was misplaced.
* **Confirm dead code via a repo-wide grep of every call form before deleting.** `figures.kpi_card` was a stale dup of `layout.kpi_card`.
* **ruff ambiguous-unicode rules (RUF001/2/3) fire on intentional text — keep them ignored, leave the literals.** Deliberate glyphs: full-width Japanese punctuation in Python literals (有/無) and scientific typography (en-dash ranges `0–100`, `×`, `σ`, thin spaces). They sit in `[tool.ruff.lint] ignore` beside `E501`/`B008`. Governs *code*; the human-prose rule below is a separate surface.
* **Human-facing prose (CLAUDE.md L26) applies ONLY to `README.md` + the `en` values of `schema/ui_strings.yaml`:** flatten em/en-dashes to ASCII hyphens, enumerate flexibly, vary comparatives. Leave `ja` UI typography, agent-facing files, and code comments/docstrings as-is. **Preserve the math minus `−` (U+2212) everywhere** (not a dash; e.g. `z(Δf) − z(Δn)`).
* **Positive framing (CLAUDE.md L11) covers self-read/prompt text incl. this file** — phrase directives as keep/always/rather-than. Factual `never`s that *describe* an invariant (IDNumber never null, etc.) are descriptions — leave them.
* **LightGBM training is byte-reproducible here; k-means archetypes are not.** A full retrain reproduces every LightGBM metric exactly (fixed `random_state` suffices), so a diff in the `outcomes` block is a *real* change. The lone churn is `archetypes.centroids` (~1e-15 BLAS thread-order noise); `git checkout models/training_metrics.json` drops it.
* **Two writers touch `training_metrics.json`** — `train.py` first, `archetypes.py` rewrites last (adds the `archetypes` key); all use `ensure_ascii=False` (raw UTF-8 JP). Run the full `train`→`archetypes` sequence (§6) before trusting or diffing it.
* **Data-quality validation must read the RAW frame (or a raw→clean diff); the loader is defensive and hides anomalies.** Out-of-range numeric/ordinal → NaN (re-parse raw vs schema `range` to recover); unmapped categorical kept as the raw token (`normalize_level` falls back to the stripped raw string, never NA) — detect those by testing the cleaned value against the level set's `display` set, NOT by looking for NaN. Pattern in `data/quality.py`.
* **`class_weight="balanced"` trades calibration for recall — treat its probabilities as uncalibrated.** The balanced magnitude head emits inflated minority probs; vs a calibrated unweighted binary head they disagree (binary P(≥C)=0.51 vs magnitude P(≥+2)=0.76). Rule: near-balanced target → omit weighting + Platt/isotonic-calibrate; imbalanced target needing weighting → surface the APS set / argmax class, not the raw probability. Also: `TreeExplainer(lgbm_binary).shap_values(X)` now returns a *list of ndarray* — take `[-1]` for the positive class; the 3-D `(n,p,2)` form takes `[:,:,-1]`.
* **"Dashboard boots 200" does NOT exercise tab content — render each touched tab in both langs.** An `INK["600"]` typo (INK has 900/700/500/300/…) crashed the Methods-tab render for four sessions yet every "boots 200" check passed (tab bodies build lazily via the `update_tab` callback). Guard: headless `render_methods('ja'/'en')` / `render_patient` / `render_simulator` + invoke new callbacks directly.
* **The admission-feature count is 30, not 32** (docs drifted; cite "the admission features" without a number). `af.feature_cols` == `len(ADMISSION_FEATURES)` == **30**: 2 demographics + 9 injury/admin + 15 ISNCSCI/AIS + 4 SCIM-ADL. Derive from `len(af.feature_cols)` at runtime.
* **A conformal prediction SET over a BINARY head degenerates — use the calibrated probability + reliability curve.** APS over {dependent,independent} gave 100% coverage with ~90–100% abstain (`{both}`) on all 18 heads: the APS nonconformity score is cumulative mass to the true class, so every misranked episode scores exactly 1.0; accuracy ~0.85 pins ~15% of scores at 1.0, dragging the 80%-quantile q→~0.98–1.0, forcing `{both}` whenever p_top<q. For K=2 the calibrated probability *is* the honest uncertainty. Reserve APS/abstention sets for K≥3 (AIS, magnitude). Verified q_hat=0.98–1.00 before removing.
* **`build_episode_frame` ends with `feat.reset_index()` ⇒ `ep` (=`af.df`) has a POSITIONAL index (0..N−1) and `KeyRecordNumber` is a COLUMN** — align any separately-built per-timepoint matrix by `ep["KeyRecordNumber"]`, NOT `.reindex(ep.index)`. Reindexing per-segment ISNCSCI matrices (which are `set_index("KeyRecordNumber")`) onto `ep.index` pulled KRNs 0..898 (real values are non-contiguous 1..1169) = a row scramble. **Trap: per-segment AUC stayed ~0.87** because label `y` and the segment's `adm_self` were pulled by the *same* wrong reindex (mutually consistent). Fix: `mtx.reindex(ep["KeyRecordNumber"].to_numpy()).set_axis(ep.index)` (AUC→0.93). Catch a co-misaligned feature+label with a BEHAVIORAL check, not AUC: a max-admission-motor patient must predict P(antigravity)≈1 and a zero-motor complete injury ≈0 (mean |ΔP|≈0.9); the scrambled model gave a flat ~0.68/0.96.
* **Claude deny-`Read()` rules are gitignore-style; a `dir/*/**` pattern over-denies — verify empirically.** `Read(/models/*/**)` also denied the direct-child `models/*_metrics.json` (`*` matches a filename, `/**` matches zero segments). Silent in *both* directions. After any edit (rules reload live) test a path that MUST block AND one that MUST stay readable. Prefer extension globs (`**/*.joblib`) for binary trees + exact file rules for dumps; a leading `/` anchors to project root (`//` = fs-absolute), a bare name matches at any depth. **The deny glob is also matched against the whole Bash command string** — any command whose text contains a denied literal is itself blocked (even `ls`/`wc` on it). Work around: query a denied JSON with `jq` (no denied substring); load a denied binary via an import that reads the path internally (a `python` heredoc importing `state.py`, which `joblib.load`s the bundle).
* **A feature that nails a STATE need not help a CHANGE / threshold-crossing target — test the transfer.** Adding a segment's own admission grade jumped segment-*state* AUC 0.5→0.93 (admission≈discharge autocorrelation); replaying it for neurological-*level descent* moved AUC ≤0.02 (reverted). A level descends only when its most-rostral impaired segment crosses a grade threshold — a recovery-dynamics event static grades don't encode. Keep the simpler model when enrichment doesn't pay.
* **Score the DISSOCIATION/mismatch between two recovery axes with the standardized contrast z(A)−z(B), NOT a regression residual.** When the axes are weakly coupled (Δneuro↔Δfunction r≈0.12–0.27) the residual ≈ the target axis itself. The z-contrast weights both equally (corr ≈±0.6 with each), is mean-0, and its sign is interpretable (D>0 = functional over-achiever). Check the score's correlation with each input axis first.
* **Before claiming a derived-target head is novel, correlate it vs the naive composition of existing heads.** Predicted dissociation D correlates ≈0.97 with simply differencing two separately-modeled axes — so the value is the *framing/surface* (one calibrated, signed dissociation + PI), not a new predictive signal. State that honestly.

## 1. Data invariants (established — rely on them)

* **Raw file** — `ALL_SCIDATA.csv` at repo root; gitignored, uncommitted.
* **`schema/raw_profile.json` stays gitignored/uncommitted** — the generated column profile is identifier-bearing (real medical-record numbers via IDNumber min/max/median; record-entry timestamps via STAMP). Purged from history (git filter-repo, 2026-06). `scripts/01_profile_raw.py` redacts `IDENTIFIER_COLS` to existence + missingness; the file stays untracked + deny-`Read()`'d (regenerate locally). No runtime code reads it.
* **Encoding** — `cp932` (Shift-JIS superset). UTF-8 silently mangles half the column names.
* **Missing sentinels** in raw: `""`, `"_"`, `"NA"` → NaN via `na_values` in `loader.py::load_raw`. `"NT"`/`"ND"` are not listed but become NaN as a side effect of `to_numeric(errors="coerce")` (numeric/ordinal) and `normalize_level()` returning `pd.NA` for unknown levels. No `_apply_missing_sentinels` helper exists.
* **Feature missingness is high and preserved into the model.** `train.py::_prep` drops only rows missing the *target* or `IDNumber`; feature NaNs flow into LightGBM (native missing). Admission missingness is the cohort norm — 糖尿病~79%, Frankel~74%, 損傷部位/保険~73%, ALLEN~66%, motor/sensory levels~30%, SCIM~22%, AIS~12%. So NaN at inference (F25 partial input) matches training, and the conformal calibration set was built under this missingness — the 80% PI stays approximately valid. The conformal half-width `q` is a fixed calibrated scalar (Mondrian-by-AIS → marginal when `AIS_ord` is blank), so it does **not** widen as input sparsens; the simulator's reliability badge is the separate completeness/OOD cue.
* **Excel booleans** — many bool-like columns arrive as the literal strings `"FALSE"`/`"TRUE"` (uppercase) → 0/1 → Y/N via the schema level mapping.
* **mFrankel/Frankel** — single combined raw column; split on slash into `mFrankel_ord` (5-grade A–E with substages) and `Frankel_ord` (5-grade).
* **Raw shape** — long format: 31,200 rows × 219 cols, 1,200 `KeyRecordNumber`s × 26 timepoint slots (`0day`,`72h`,`2w`,`4w`,`6w`,`2m..11m`,`1y..10y`,`discharge`). Perfectly rectangular — every episode has a row at every slot.
* **Ghost-episode filter** — 301 of 1,200 raw episodes are pure placeholder rows (`IDNumber` null AND every admission feature null AND every outcome null). `build_analysis_dataset()` drops them via `_identify_ghost_episodes(ep, ADMISSION_FEATURES)`. **Post-filter universe: 899 episodes / 866 unique patients; long frame = 23,374 rows (899×26).**
* **Partial-id orphans (27 episodes)** — admission features but null `IDNumber`. Survive the ghost filter, excluded from training by `dropna(subset=["IDNumber", outcome])` in `_prep()` and from the picker by `list_patient_options(ep)`; they still feed cohort aggregates. (9 discharge SCIM, 10 discharge AIS, 14 `LOS_days`.)
* **IDNumber 1-off (raw 867 → clean 866)** — `KeyRecord 446` has literal `'6641/10/15'` (malformed date) in the ID field; schema `IDNumber: numeric` → `to_numeric(errors="coerce")` NaN's it.
* **Outcome cardinality (899-episode universe)** — `y_discharge_scim`: 507; `y_discharge_ais`: 638; `y_discharge_wisci`: **50 — too sparse, stays dropped**; `LOS_days`: 682.
* **Subscale outcomes share SCIM-total's universe** — `y_discharge_scim_{self_care,resp_sphincter,mobility}` from the same `discharge` rows (n=507; training n=498 after dropping IDNumber-null orphans). Ranges: self-care 0–20, resp/sphincter 0–40, mobility 0–40, total 0–100.
* **AIS class imbalance** — D dominates (~59%); B and E ~5% each. LightGBM `multiclass` with `class_weight="balanced"`, else B/E are never predicted.
* **LOS distribution** — heavy right tail (median≈140d, max≈790d). Modelled on `log1p`; conformal q computed in log-space, back-transformed.
* **`LOS_days` is right-censored by recency** — `入院期間` is recorded only at discharge: complete through 2022, ~28/78 in 2023, **zero for `BusinessYear` ≥ 2024** (SCIM/AIS discharge outcomes extend to 2025). F24's temporal backtest covers only 2020–2023 origins for LOS.
* **`BusinessYear` is an episode-frame passport, never a feature** — first-non-null per episode (100% populated, 2014–2025). Absent from `ADMISSION_FEATURES`, so never in `feature_cols`; it exists only as the temporal-split key for F24.
* **`mFrankel_Frankel` is a packed pair column** — missing marker `_/_`; valid values include mismatched pairs (`C1/C2`). Validate through the split `mFrankel_ord`/`Frankel_ord`, not the pair string.
* **`ALLEN分類` raw values are messy** — full-width Roman stages (`DE-Ⅱ` vs ASCII `DE-2`), out-of-enum stages (`VC-3`), misplaced bony/level tokens — ~442 rows fail to normalize via the `allen` level set. It is a model feature, so this is a real coverage gap (add Roman-numeral `raw_aliases`, or treat as noisy).
* **PinPrick sensory cells carry asterisk-annotated scores (`1*`, `0*`)** — the ISNCSCI "non-SCI cause" convention. `to_numeric` drops them to NaN, so the annotation is silently lost on load.

## 3. Model conventions (design contracts; metrics live in `training_metrics.json`)

### Production heads (`train.py` artifacts — byte-reproducible)
* **Random state** `20260518` (also in `training_metrics.json`). **Group split by `IDNumber`** (patient, not row) — no same-patient leakage. LightGBM training is byte-reproducible (§0b); only k-means archetype centroids carry ~1e-15 noise.
* **Outcome registry** — `models/outcomes.py::OUTCOMES`, ordered tuple of `OutcomeSpec`: 6 production heads (4 SCIM + AIS + LOS) + the 5 G9 Δ heads. `train.py` iterates it; the dashboard imports the same list. **To add an outcome: extend `OUTCOMES`, put its target on the episode frame, add a `ui_strings.yaml` `outcome_{key}` entry — training/conformal/SHAP/simulator/patient/insights/Methods all extend automatically, zero new code/callbacks** (G9 added none).
* **Per-outcome artifacts** — `models/{spec.key}/` holds `lgbm_median/p10/p90.joblib` (regression) or `lgbm_multiclass.joblib` (AIS) + `feature_spec.joblib` (`conformal_q_*`, `transform`, `clip_min/max`) + `shap_test.joblib`. Top-level `models/feature_spec.joblib` is the *shared* feature universe (feature_cols, ranges, categories). `training_metrics.json` = `{"outcomes":{...}, "outcome_keys":[...]}`. Any top-level `lgbm_*`/`shap_*.joblib` are stale single-outcome-era debris (unloaded).
* **Conformal PI (regression)** — (1−α)-quantile of `|y−ŷ|` on a held-out calibration fold, on the *transformed* scale (identity for SCIM/AIS, log1p for LOS), back-transformed, clipped to `[clip_min, clip_max]`. **Required** — LightGBM quantile heads alone give ~0.41 coverage on SCIM total. At inference the PI is the **union** of conformal and raw-quantile intervals (`lo=min(lo_conf,lo_q10)`, `hi=max(hi_conf,hi_q90)`) — user sees the more conservative bound.
* **Mondrian conformal** — per-AIS-grade and per-paralysis-class q, in `feature_spec.joblib["conformal_q_by_group"]` = `{"ais":{...},"paralysis":{...},"marginal":q,"min_n":8}`. Groups with < `MONDRIAN_MIN_N=8` calibration samples omitted; inference falls back AIS→paralysis→marginal. AIS-C gets widest PI, AIS-D tightest for SCIM (clinically correct, hidden by the marginal).
* **AIS multiclass head** — classes encoded by severity A=0…E=4 (0-based model index; the `AIS_ord` *data* column is 1–5 via `constants.AIS_LETTER_TO_ORD`), so `predict_proba` cols + SHAP last axis are ordinally sorted. Metrics: accuracy, quadratic-weighted κ, MAE-on-ordinal-code.
* **APS conformal sets (AIS)** — calibration fold from dev; APS nonconformity = cumulative prob mass to include the true class; threshold at ⌈(n+1)·0.8⌉ quantile → `q_hat`. Mondrian variants under `aps_q_hat` + `aps_q_by_group`. Inference: sort class probs desc, accumulate until cumsum ≥ resolved q_hat. **Coverage runs conservative (~99% vs 80% target) because K=5 makes APS scores discrete — expected** (avg set ≈2.8).
* **LOS (log1p) head** — same regression machinery; `y` log1p'd, conformal q in log-space; predictions/PI back-transformed via `expm1`, clipped `[0,∞)`. Metrics in raw days. LOS is hard in- and out-of-time (R²≈0.2) and right-censored (§1).
* **TreeSHAP** on the held-out **test set only** (training SHAP is optimistic), cached in `shap_test.joblib`; multiclass cache is 3-D `(n,p,K=5)`, AIS axis last (insight engine slices by `class_idx`). Also persists `shap_interaction` (regression `(n,p,p)`, multiclass `(n,p,p,K)`; diagonal=main effect, off-diag=symmetric pairwise; cats pre-encoded to int codes per §0b; `training_metrics.json` stores `global_interaction_top25`; dominant interaction = age × motor) and `y_test`/`y_pred`(/`y_pred_proba`) for the Methods calibration visuals.
* **Holm correction** — running **max** over sorted p×(n−k+1), not min (§0b).
* **Trajectory models** — 9 independent LightGBM regressions of SCIM-total at 72h…6m from the same admission features. `models/trajectory/bundle.joblib` (`timepoints`,`models`,`conformal`,`clip_min/max`); each timepoint median/p10/p90 + Mondrian q. No SHAP. Admission features most predictive of ~1-month, least of ~5-month; conformal q widens monotonically. Value is trajectory *shape*, not per-timepoint accuracy.
* **Recovery archetypes** — k-means (k=3 by silhouette) on predicted 10-point trajectories (9 timepoints + discharge), z-scored per timepoint. Ordered by discharge SCIM: 0 limited / 1 gradual / 2 rapid. `data/archetypes.py::assign_single()` assigns at runtime. `models/archetypes/archetypes.joblib`. Primary separators: AIS grade, age.
* **Temporal validation (`models/temporal.py`, F24)** — out-of-time rolling-origin backtest, **diagnostic only**; writes `models/temporal_metrics.json`, never touches `train.py` artifacts. Expanding window, test years 2020–2025, train=`BusinessYear < T`, group-safe (test episodes whose patient appears in the past are dropped + counted `n_dropped_overlap`). Reuses train/conformal helpers; deltas: temporal cut, **marginal** conformal/APS (per-origin folds too small for Mondrian); reported coverage is conformal-only. Finding: SCIM lose only ΔR²≈−0.04…−0.08 OOT (coverage ≈nominal); AIS robust (APS ~0.99); LOS hard both ways.

### Diagnostic/inference model families (G1–G11) — shared pattern
**State once, applies to G1/G2/G4/G6/G7/G8/G10/G11 unless noted:** grouped-5-fold-CV-by-`IDNumber` → **out-of-fold (OOF)** predictions drive every metric, the Platt (sigmoid) calibrator for binary heads, and a **cross-conformal** APS q for multiclass (valid small-n analogue of production's single split-conformal); heads **refit on the full cohort** reusing the OOF calibrator/q (conservative). SHAP = **descriptive in-sample** drivers (not OOS). Binary plumbing (`_typed_X`/`_oof_binary`/`_fit_platt`/`_apply_platt`/`_refit`/`_calibration_curve`/`_shap_top`) is **imported verbatim from `conversion.py`**. Each module writes a **tracked identifier-free `models/<name>_metrics.json` + git-ignored `models/<name>/bundle.joblib`** and **never touches `train.py` artifacts (byte-repro verified)**. `_apply_platt` is **mirrored inline in `compute.py`** so the dashboard never imports the model module (it pulls shap via `train.py`/`conversion.py`). Bundle/metrics shapes are documented inline atop each module. `MIN_COHORT=120` (where applicable). Patient cards that are **admission-grade-gated** override `AIS_ord` with the episode's *raw* admission grade before inference (cohort membership is undefined when the grade is unrecorded, ~12%); **non-gated** cards (G7/G8/G11) use `episode_row_for_model`'s imputed row as-is, no override.

* **G1 landmark (`models/landmark.py`)** — per landmark L∈{72h,2w,4w,6w,2m,3m} refit the **discharge** outcome on admission features **+ a LOCF observation block** (`LANDMARK_COLS`: SCIM total+3 subscales, AIS_ord, UEMS, LEMS, TotalMotor, LightTouch, PinPrick, each carried forward as last non-null at a timepoint ≤ L, prefixed `L_`; landmark feats=40 vs baseline 30). **Eligibility = still-admitted risk set** (episode enters L only if it has an observation at an intermediate timepoint with order-index ≥ L; `discharge` excluded from `TIMEPOINT_ORDER`) — avoids immortal-time/leakage. Each landmark paired with an **admission-only baseline refit on the identical eligible cohort+split** so the delta isolates the value of observation. Marginal conformal/APS. Persists **both heads per (outcome,L)** under `by_landmark[L]={"baseline","landmark"}` + `timepoint_order`. Finding: SCIM ΔR² +0.04(72h)→+0.30(3m), PI half-width ≈23→13.
* **G2 value-of-information (`landmark.py` + `compute.landmark_voi`)** — per (outcome,L) also fits 10 **single-add heads** (admission + exactly one `L_<measure>`, 31 feats), each its own marginal q; metrics under `by_landmark[L]["single"][measure]`, plus `lm_value_summary[L][measure]={median,q25,q75,n}` for cohort-median imputation. `landmark_voi(key,L,X_base,observed)` ranks measures for one patient: real value if present (`observed`, realized gain) else cohort median (`prescriptive`, hypothetical). **CRUX — marginal q ⇒ a single-add head's PI half-width is a per-MEASURE constant, not per-patient**: for identity outcomes (SCIM/AIS) `d_halfwidth` is patient-independent (only the point estimate personalizes); **log1p LOS is the exception** (constant q in log-space, `expm1` scales it ⇒ patient-specific). So VOI bars rank a population quantity for SCIM/AIS, a personal one for LOS — by design.
* **G3 phenotyping (`data/phenotypes.py` + `models/phenotypes.py`)** — **multivariate growth mixture model** (mixture of linear mixed models) over *observed* early SCIM-total + total-motor trajectories jointly (contrast §3 archetypes = k-means on *predicted* curves). `y_{i,m}(t)=β_{k,m}·φ(t)+b_{i,m}·ψ(t)+ε`; φ=poly degree D (BIC ∈{1,2}), ψ=[1,t]. **Class-invariant `G` (block-diag) + `sigma2`** (avoids variance-collapse; `V_i=Z_i G Z_i'+R_i` class-independent, built once/individual/EM-iter; only mean is class-specific — distinct from LCGA's G=0). **Native missingness** (each individual contributes only its observed cells) = the win over k-means-on-imputed. EM (log-sum-exp E; ECM M: resp-weighted GLS β + Laird-Ware variance components), multi-restart, K×degree by BIC + min-class-share floor. Fit: **K=5, degree=2** (590 episodes ≥3 SCIM points). **CRUX — degree-2 extrapolation traps:** means valid only over each class's *observed support* (an early-discharge class dives to SCIM −1277 at 6m). `class_support(...min_coverage=0.20)`→last window index where ≥20% still observed; (a) blank curves past support (+clip [0,100]); (b) `order_by_discharge` ranks by clipped SCIM at the **latest universally-supported** timepoint (raw 6m would mislabel the best early-discharge recoverer as worst). **Invariant: full-window membership reproduces the bundle's stored `posterior` row byte-for-byte for an in-cohort episode** (verified 0.0); generalizes to out-of-cohort episodes (fresh `predict_proba`). `compute.predict_phenotype_membership`, `phenotype_cutoff_options`.
* **G4 AIS conversion (`models/conversion.py`)** — admission→discharge AIS *transition* (not absolute `y_discharge_ais`), admission grade kept as feature (conversion is admission-gated: ceiling at D, none at E). Heads: 2 **calibrated binary** (`motor_incomplete` A/B→≥C; `ambulatory` A–C→≥D) + **ordinal magnitude** multiclass {0,+1,≥+2} (`MAG_CAP=2`) on the A–D room-to-improve cohort (deterioration→class 0). **CRUX — binary vs magnitude are NOT comparable:** binary heads use no `class_weight` (near-balanced→calibrated); magnitude uses `class_weight="balanced"` (uncalibrated, inflated minority probs) — they disagree (adm A: binary P(≥C)=0.51 vs magnitude P(≥+2)=0.76). UI: binary = the calibrated probability, magnitude = APS set / argmax (never a competing probability). **Requires `AIS_ord`** (gates: endpoint applies only if adm grade ∈ its `adm_grades`; magnitude A–D only; `ais_ord=None`→"needs-grade" prompt). Findings: `ambulatory` AUC≈0.87; `motor_incomplete` AUC≈0.62 (admission poorly predicts complete→incomplete — a real clinical reality); magnitude κ≈0.49.
* **G6 AIS multi-state (`models/multistate.py`)** — AIS-grade *trajectory* over the dense `0day..6m` grid. (1) **Population time-inhomogeneous Markov** over 5 states (A=1..E=5): per step the empirical `P_k` from **pairwise-complete** episodes, zero-departure row → identity (mass-conserving); forward-multiply from an admission point mass, **stratified by admission grade**, for occupancy, first-passage conversion-to-≥X (≥X made absorbing ⇒ monotone; labels `improve`=≥adm+1, `ge_C`, `ge_D`), median day to improvement, expected days/state (`np.trapezoid`, NOT numpy-1 `np.trapz`). **CRUX — apparent regressions are real** (B→A, D→C reassessment/inter-rater noise → backward mass; surface honestly; small-n cells expose it). (2) **Covariate improve-by-6m binary** (P(≥1-grade improvement) on A–D cohort with ≥`MIN_WINDOW_OBS=2` in-window obs; no class_weight→calibrated). AUC≈0.90, Brier≈0.12. **Clinical finding — improve base rate is NON-monotone in admission grade: B highest ~85%, C ~74%, A ~64%, D lowest ~15%** (D-improvement means reaching **E**=full-normal; UI frames D's head as P(D→E), `to_e=True`). **Cohort, not personalized** — occupancy/conversion/sojourn depend only on admission grade; only the `improve` prob + own-trajectory overlay differ between same-grade patients. First-passage dedup: `improve`(=adm+1) coincides with ≥C/≥D for B/C admissions — map to distinct target grades to avoid duplicate curves. E admission = `at_ceiling` (occupancy only).
* **G7 functional-independence (`models/independence.py`)** — **18 calibrated binary heads**, P(**functional independence** at discharge: aids/devices allowed, no human assistance) per SCIM-III ADL item (Respiration excluded, 95% independent → too imbalanced). **Threshold table (`independence.py::ITEMS`, load-bearing):** feeding/bathing/dressing/grooming **≥2**; bladder **≥9** / bowel **≥8**; toilet **≥3**; bed-mobility =6 and the four transfers (bed↔WC, WC↔toilet, WC↔car, ground↔WC) at **top score** (scale lumps human-assist with device-use); the 3 **walking** items (indoors/moderate/outdoors) **≥4** = ambulates with-or-without aids (wheelchair-independence excluded — distinct ambulation milestones). Domains: self_care(6)/sphincter(3)/mobility(5)/ambulation(4). **CRUX — uncertainty is the calibrated prob + reliability curve, NOT a conformal set (binary APS degenerates, §0b — removed).** Findings: mean AUC≈0.905 (range 0.88–0.93); **expected-independent count** (Σ of 18 probs) rises monotonically with admission AIS (A≈0.9→B≈6→C≈13→D/E≈17 of 18); outdoor walking hardest (P≈0.36 even at AIS-C). **NOT grade-gated** (no real-grade override). **CRUX — `AIS_ord` alone is a weak driver (TotalMotor/UEMS dominate SHAP): a Simulator profile with only an admission grade is flat (~8.5/18), NOT grade-differentiated; the A→E spread emerges only when correlated motor/sensory move with the grade** (honest under missingness, consistent with F25).
* **G8 topography (`models/topography.py`)** — per-ISNCSCI-segment recovery atlas: for **all 132 segments** (20 motor + 56 LT + 56 PP, L/R), calibrated P(milestone at discharge): motor **P(grade≥3)**=antigravity, sensory **P(grade≥1)**=preserved sensation. **Method = 132 INDEPENDENT calibrated binary heads (NOT the low-rank model first chosen). CRUX — a diagnostic settled it: the 30 aggregate features predict a *specific* segment near-chance (OOF AUC 0.43–0.63); the dominant predictor is the segment's OWN admission grade (diagonal, not cross-segment), and the discharge matrix isn't low-rank. Adding `adm_self` (the segment's own admission grade, LOCF, still admission-only) lifts AUC to ~0.93.** **Alignment invariant: align per-segment matrices to `ep` by `ep["KeyRecordNumber"]`, NOT `ep.index` (§0b) — co-misalignment keeps AUC high while scrambling context+groups; verify with the behavioral check.** 12 degenerate heads (high-cervical C2/C3 LT+PP, < `MIN_MINORITY=12`) carry constant base-rate, no model. Findings: mean OOF AUC motor 0.94/LT 0.93/PP 0.91. **The expected-antigravity count is driven by injury LEVEL, not AIS — it is NON-monotone in admission AIS (A 12.7, B 16.3, C 14.5, D 15.2, E 15.8) — a "rises with AIS" sanity check is WRONG here** (unlike SCIM/independence). **NOT grade-gated.**
* **G9 Δ score-recovery (G9 — `outcomes.py` registry, NOT a standalone module)** — predicts admission→discharge *change* in 5 ISNCSCI summary scores: ΔUEMS, ΔLEMS, Δtotal-motor, Δlight-touch, Δpin-prick (`delta_{uems,lems,totalmotor,lighttouch,pinprick}`), the canonical SCI-trial endpoint. **Pure production-registry extension** (two edits: 5 `OutcomeSpec`s into `OUTCOMES`; 5 `y_delta_*` targets in `build_episode_frame` = discharge-slot − the same first-non-null admission feature; NaN either side→NaN, dropped). `train.py` untouched; reuses SCIM regression machinery (median/p10/p90 + Mondrian conformal + SHAP, identity transform). **No leakage** (label is the delta). **Negative-range invariant: `clip_min < 0`** (±50 motor / ±100 total-motor / ±112 sensory) so deterioration is representable; `clip_min < 0` is also the UI's unique **Δ-head flag** → signed point/PI (`+11`/`−3`), drops the "/ max" suffix, draws a dotted "no change" zero-line spanning ±range. **Behavioral gate** (not R²): at-ceiling patient predicts Δ≈0, high-headroom large +Δ (ΔUEMS +1.2 vs +29.1). Finding: motor recovery more predictable than sensory; dominant SHAP shifts to `mFrankel_ord × <matching baseline score>`. Heavier diagnostic surfaces (F24 temporal, G1/G2 landmark) **gracefully skip** Δ outcomes (`if not info: continue`) — accepted boundary.
* **G10 level descent (`models/level_descent.py`)** — admission→discharge change in 5 ISNCSCI neurological *levels* (NLI + bilateral motor + bilateral sensory), complement to G9's Δ scores. Cord-level ordinal **C1=0..S45=28** (smaller=more rostral=more severe); **Δ>0 = caudal descent = improvement**. **`INT→INT_ORD=29` ceiling:** full recovery (raw `INT`, which the loader maps to NaN) lifted to ord 29 at discharge so a cure is the Δ ceiling, not a dropped row; admission baseline uses loader `*_ord` (INT/missing→NaN) so already-intact admissions drop (only ord-0..28 enter). **2 heads/level (10):** calibrated binary P(Δ≥1) (no class_weight, Platt) + ordinal magnitude {0,+1,≥+2} (balanced, APS) — same binary-vs-magnitude CRUX as G4. Findings: **only NLI well-predicted (AUC≈0.73); bilateral motor/sensory AUC≈0.62–0.63 — a genuine admission-signal ceiling** (single-boundary threshold-crossing is noisy). **Report the median** (INT→29 inflates NLI mean +3.1 vs median +1). **Enrichment rejected**: G8-style concat of per-segment admission grades left AUC flat-to-worse (§0b state-vs-threshold-crossing lesson).
* **G11 neuro-functional dissociation (`models/dissociation.py`)** — predicts the *mismatch* between neurological (Δ ISNCSCI score) and functional (Δ SCIM subscale) recovery. **Score `D = z(Δfunction) − z(Δneuro)`** (D>0 = functional over-achiever / compensation; D<0 = neurology outpaces function). **Contrast, not a regression residual** — the axes are weakly coupled (Pearson r≈0.21/0.12/0.27) so a residual would collapse to the functional axis (§0b). 3 domain-paired axes: `uems_selfcare`, `lems_mobility` (the r≈0.12 decoupling: lower-limb strength barely predicts mobility independence — wheelchair skills), `totalmotor_total`. **2 heads/axis: over_achiever** = calibrated binary P(D>0); **magnitude** = continuous regression on D + marginal cross-conformal 80% PI. **Inverse of the G4/G10 CRUX — magnitude is a regression (no class weighting) so the two heads ARE honestly comparable** (sign(magnitude)≈binary direction); surface binary=prob, magnitude=signed point+PI. All axes n=438. Findings: binary AUC 0.93–0.95, magnitude R² 0.69–0.76; dominant driver = the NEURO-axis baseline score (ceiling/room), age #2. **Honest-novelty caveat (§0b): predicted-D ≈0.97-correlates with naively differencing two separately-modeled axes — value is the framing (one calibrated signed dissociation + PI), not a new mechanism. Star-placement CRUX (UI): a 1-D D has no 2-D position without an external anchor — x = the patient's predicted Δneuro from the matching G9 head, vertical gap = sd_f·D.** **NOT grade-gated.**

## 4. Dashboard conventions

* **Module layout** (`src/rehab_sci/dashboard/`) — browse the per-file symbol inventory via Serena LSP (`get_symbols_overview`); only the non-obvious contracts live here:
  - `state.py` — all startup globals, loaded once; depends only on theme + data/model layers (no other dashboard module imports it transitively).
  - `compute.py` — pure (model inference, conformal-q, APS, SHAP, row-prep), **no Dash/Plotly**. Landmark inference lives here: `predict_landmark` (paired baseline+landmark heads from `LANDMARK_BUNDLE`), `landmark_observed_for_episode` (real LOCF block on the still-admitted risk set), `episode_landmark_eligibility` (per-L still-admitted gate).
  - `reliability.py` — pure; `assess_input(X, bundle, feature_spec)` → importance-weighted completeness + OOD (range violations / atypicality vs `feature_spec['ranges']` q05–q95) for the simulator partial-input badge.
  - `layout.py` — the shared *prediction* figures (`fig_shap_local`, `fig_prediction_interval`, `fig_class_probabilities`) + landmark compare (`fig_landmark_compare` + `landmark_readout`), shared by simulator & patient cards — **not** in the figures package.
  - `figures/` — a **package** split by tab (`overview`,`insights`,`patient`,`methods`,`simulator`) + `_common`; every public name re-exported via `figures/__init__.py` `__all__` (so `from rehab_sci.dashboard import figures as fg` + every `fg.fig_*` are unchanged). Add a figure: put it in its tab submodule and append the name to `__all__`.
  - `tabs/*` = per-tab layout + `@callback`s; `app.py` = entry + chrome callbacks.
* **Dependency graph (acyclic):** `state`→data/model; `compute`→`state`; `layout`→`state`,`compute`; `tabs/*`→`state`,`compute`,`layout`,`figures`; `app`→`tabs/*`. Importing a tab triggers `@callback` registration (Dash function-level `@callback` registers globally — no `@app.callback`).
* `dcc.Store("lang-store")` holds `"ja"`/`"en"`; most text callbacks take it as `Input` for instant swaps. **Exception:** `update_overview_content` takes it as `State` to avoid a race with `update_tab` (both fire on lang change — `State` makes overview fire *after* `update_tab` rebuilds the filter components with new-lang labels).
* Pattern-matched simulator inputs use IDs `{"type":"num"/"cat","col":<raw>}` with `dash.ALL`; order fixed by `feature_spec['feature_cols']`. Numeric fields are **clearable `dcc.Input(number)`** (not sliders); the form opens **blank** (F25) — `collect_sim_inputs` leaves blanks as NaN (no imputation); *Fill cohort defaults* / *Clear all* set every value via one pattern-matching value-Output callback; What-if mode prefills from the reference episode. (`simulate` returns 5 outputs incl. `sim-reliability`.)
* Plotly template `"medical"` (registered in `theme.py`). Palettes `PALETTE_CATEGORICAL`, `PALETTE_AIS` (A→E cool→warm), `PALETTE_PARA`, `PALETTE_ARCHETYPE` — use them for every chart. Japanese rendering needs the font stack `"Hiragino Sans","Noto Sans JP","Yu Gothic UI"` in both Plotly and CSS.
* `dcc.Store("patient-ref")` (session-scoped) carries the What-if reference `{id_number,key_record,features,outcomes,trajectory}`; `update_tab` reads it as `State` to prefill simulator defaults, `simulate` reads it as `State` for the reference overlay.

## 5. Known gotchas

* **`IntCastingNaNError` on `IDNumber`** — patients with no admission row produce NaN IDs; `dropna(subset=[outcome,"IDNumber"])` then cast `float64→int64`.
* **Stale dashboard process** — `kill <PID>` stops only the `uv run` wrapper; the Python child keeps serving old code. `pkill -f 'rehab_sci.dashboard.app'` **self-matches the calling shell** (the pattern is in its own argv) → the shell is signalled too (the exit-144 / signal-16 report). So: (1) it still kills the dashboard, fine as the **last** statement of a cleanup-only command; (2) **put it after anything else you still need to run**, or the shell dies at signal 16 first (empty logs, nothing listening — looks like a boot failure but is not). Prefer: launch via the Bash tool's background mode (no pkill), poll readiness in a **separate** command containing no matching literal (curl only), stop by port `fuser -k 8050/tcp` (never self-matches). Verified boot = `curl -s -o /dev/null -w '%{http_code}' http://127.0.0.1:8050/` → 200, plus `/_dash-dependencies` for the callback graph.
* **pandas fragmentation warning** when adding many columns serially — batch via `pd.concat([df, new_cols_df], axis=1)` (loader already does this).
* **`@dataclass(frozen=True)` + dict fields** breaks under `@lru_cache` on instance methods (unhashable) — `Schema` stays a plain `__slots__` class.
* **`kaleido<1`** has no Linux x86_64 wheel under the current resolver — keep `kaleido>=1.0,<2`. Plotly→PNG needs Chrome (`kaleido.get_chrome_sync()` at first use).
* **Packaging** — a real hatchling uv package (`[tool.hatch.build.targets.wheel] packages=["src/rehab_sci"]`); `uv sync` installs it editable, so `uv run python -m rehab_sci.*` needs no `PYTHONPATH`.
* **Background dashboard from a bash one-liner** — `nohup … &` inside the harness wrapper may not survive wrapper exit (exit 144 = SIGTERM bookkeeping); use the Bash tool's `run_in_background: true`, or make the launch the *last* statement.
* **`Sunburst(branchvalues="total")` requires parent value = sum of children** — parent `0` renders blank silently. Prefer accumulating leaf counts into every ancestor (hover shows true subtotals) over `branchvalues="remainder"`. (The injury chart later moved to a treemap.)
