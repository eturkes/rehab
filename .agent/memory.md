* **The admission-feature count is 30, not 32** (docs drifted; cite "the admission features" without a number). `af.feature_cols` == `len(ADMISSION_FEATURES)` == **30**: 2 demographics + 9 injury/admin + 15 ISNCSCI/AIS + 4 SCIM-ADL. Derive from `len(af.feature_cols)` at runtime.
* **A conformal prediction SET over a BINARY head degenerates ‚Äî use the calibrated probability + reliability curve.** APS over {dependent,independent} gave 100% coverage with ~90‚Äì100% abstain (`{both}`) on all 18 heads: the APS nonconformity score is cumulative mass to the true class, so every misranked episode scores exactly 1.0; accuracy ~0.85 pins ~15% of scores at 1.0, dragging the 80%-quantile q‚Üí~0.98‚Äì1.0, forcing `{both}` whenever p_top<q. For K=2 the calibrated probability *is* the honest uncertainty. Reserve APS/abstention sets for K‚â•3 (AIS, magnitude). Verified q_hat=0.98‚Äì1.00 before removing.
* **`build_episode_frame` ends with `feat.reset_index()` ‚áí `ep` (=`af.df`) has a POSITIONAL index (0..N‚àí1) and `KeyRecordNumber` is a COLUMN** ‚Äî align any separately-built per-timepoint matrix by `ep["KeyRecordNumber"]`, NOT `.reindex(ep.index)`. Reindexing per-segment ISNCSCI matrices (which are `set_index("KeyRecordNumber")`) onto `ep.index` pulled KRNs 0..898 (real values are non-contiguous 1..1169) = a row scramble. **Trap: per-segment AUC stayed ~0.87** because label `y` and the segment's `adm_self` were pulled by the *same* wrong reindex (mutually consistent). Fix: `mtx.reindex(ep["KeyRecordNumber"].to_numpy()).set_axis(ep.index)` (AUC‚Üí0.93). Catch a co-misaligned feature+label with a BEHAVIORAL check, not AUC: a max-admission-motor patient must predict P(antigravity)‚âà1 and a zero-motor complete injury ‚âà0 (mean |ŒîP|‚âà0.9); the scrambled model gave a flat ~0.68/0.96.
* **Claude deny-`Read()` rules are gitignore-style; a `dir/*/**` pattern over-denies ‚Äî verify empirically.** `Read(/models/*/**)` also denied the direct-child `models/*_metrics.json` (`*` matches a filename, `/**` matches zero segments). Silent in *both* directions. After any edit (rules reload live) test a path that MUST block AND one that MUST stay readable. Prefer extension globs (`**/*.joblib`) for binary trees + exact file rules for dumps; a leading `/` anchors to project root (`//` = fs-absolute), a bare name matches at any depth. **The deny glob is also matched against the whole Bash command string** ‚Äî any command whose text contains a denied literal is itself blocked (even `ls`/`wc` on it). Work around: query a denied JSON with `jq` (no denied substring); load a denied binary via an import that reads the path internally (a `python` heredoc importing `state.py`, which `joblib.load`s the bundle).
* **A feature that nails a STATE need not help a CHANGE / threshold-crossing target ‚Äî test the transfer.** Adding a segment's own admission grade jumped segment-*state* AUC 0.5‚Üí0.93 (admission‚âàdischarge autocorrelation); replaying it for neurological-*level descent* moved AUC ‚â§0.02 (reverted). A level descends only when its most-rostral impaired segment crosses a grade threshold ‚Äî a recovery-dynamics event static grades don't encode. Keep the simpler model when enrichment doesn't pay.
* **Score the DISSOCIATION/mismatch between two recovery axes with the standardized contrast z(A)‚àíz(B), NOT a regression residual.** When the axes are weakly coupled (Œîneuro‚ÜîŒîfunction r‚âà0.12‚Äì0.27) the residual ‚âà the target axis itself. The z-contrast weights both equally (corr ‚âà¬±0.6 with each), is mean-0, and its sign is interpretable (D>0 = functional over-achiever). Check the score's correlation with each input axis first.
* **Before claiming a derived-target head is novel, correlate it vs the naive composition of existing heads.** Predicted dissociation D correlates ‚âà0.97 with simply differencing two separately-modeled axes ‚Äî so the value is the *framing/surface* (one calibrated, signed dissociation + PI), not a new predictive signal. State that honestly.

## 1. Data invariants (established ‚Äî rely on them)

* **Raw file** ‚Äî `ALL_SCIDATA.csv` at repo root; gitignored, uncommitted.
* **`schema/raw_profile.json` stays gitignored/uncommitted** ‚Äî the generated column profile is identifier-bearing (real medical-record numbers via IDNumber min/max/median; record-entry timestamps via STAMP). Purged from history (git filter-repo, 2026-06). `scripts/01_profile_raw.py` redacts `IDENTIFIER_COLS` to existence + missingness; the file stays untracked + deny-`Read()`'d (regenerate locally). No runtime code reads it.
* **Encoding** ‚Äî `cp932` (Shift-JIS superset). UTF-8 silently mangles half the column names.
* **Missing sentinels** in raw: `""`, `"_"`, `"NA"` ‚Üí NaN via `na_values` in `loader.py::load_raw`. `"NT"`/`"ND"` are not listed but become NaN as a side effect of `to_numeric(errors="coerce")` (numeric/ordinal) and `normalize_level()` returning `pd.NA` for unknown levels. No `_apply_missing_sentinels` helper exists.
* **Feature missingness is high and preserved into the model.** `train.py::_prep` drops only rows missing the *target* or `IDNumber`; feature NaNs flow into LightGBM (native missing). Admission missingness is the cohort norm ‚Äî Á≥ñÂ∞øÁóÖ~79%, Frankel~74%, ÊêçÂÇ∑ÈÉ®‰Ωç/‰øùÈô∫~73%, ALLEN~66%, motor/sensory levels~30%, SCIM~22%, AIS~12%. * **Excel booleans** ‚Äî many bool-like columns arrive as the literal strings `"FALSE"`/`"TRUE"` (uppercase) ‚Üí 0/1 ‚Üí Y/N via the schema level mapping.
* **mFrankel/Frankel** ‚Äî single combined raw column; split on slash into `mFrankel_ord` (5-grade A‚ÄìE with substages) and `Frankel_ord` (5-grade).

## 3. Model conventions (design contracts; metrics live in `training_metrics.json`)

### Production heads (`train.py` artifacts ‚Äî byte-reproducible)
* **Random state** `20260518` (also in `training_metrics.json`). **Group split by `IDNumber`** (patient, not row) ‚Äî no same-patient leakage. LightGBM training is byte-reproducible (¬ß0b); only k-means archetype centroids carry ~1e-15 noise.
* **Outcome registry** ‚Äî `models/outcomes.py::OUTCOMES`, ordered tuple of `OutcomeSpec`: 6 production heads (4 SCIM + AIS + LOS) + the 5 G9 Œî heads. `train.py` iterates it; the dashboard imports the same list. **To add an outcome: extend `OUTCOMES`, put its target on the episode frame, add a `ui_strings.yaml` `outcome_{key}` entry ‚Äî training/conformal/SHAP/simulator/patient/insights/Methods all extend automatically, zero new code/callbacks** (G9 added none).
* **Per-outcome artifacts** ‚Äî `models/{spec.key}/` holds `lgbm_median/p10/p90.joblib` (regression) or `lgbm_multiclass.joblib` (AIS) + `feature_spec.joblib` (`conformal_q_*`, `transform`, `clip_min/max`) + `shap_test.joblib`. Top-level `models/feature_spec.joblib` is the *shared* feature universe (feature_cols, ranges, categories). `training_metrics.json` = `{"outcomes":{...}, "outcome_keys":[...]}`. Any top-level `lgbm_*`/`shap_*.joblib` are stale single-outcome-era debris (unloaded).
* **Conformal PI (regression)** ‚Äî (1‚àíŒ±)-quantile of `|y‚àí≈∑|` on a held-out calibration fold, on the *transformed* scale (identity for SCIM/AIS, log1p for LOS), back-transformed, clipped to `[clip_min, clip_max]`. **Required** ‚Äî LightGBM quantile heads alone give ~0.41 coverage on SCIM total. At inference the PI is the **union** of conformal and raw-quantile intervals (`lo=min(lo_conf,lo_q10)`, `hi=max(hi_conf,hi_q90)`) ‚Äî user sees the more conservative bound.
* **Mondrian conformal** ‚Äî per-AIS-grade and per-paralysis-class q, in `feature_spec.joblib["conformal_q_by_group"]` = `{"ais":{...},"paralysis":{...},"marginal":q,"min_n":8}`. Groups with < `MONDRIAN_MIN_N=8` calibration samples omitted; inference falls back AIS‚Üíparalysis‚Üímarginal. AIS-C gets widest PI, AIS-D tightest for SCIM (clinically correct, hidden by the marginal).
* **AIS multiclass head** ‚Äî classes encoded by severity A=0‚Ä¶E=4 (0-based model index; the `AIS_ord` *data* column is 1‚Äì5 via `constants.AIS_LETTER_TO_ORD`), so `predict_proba` cols + SHAP last axis are ordinally sorted. Metrics: accuracy, quadratic-weighted Œ∫, MAE-on-ordinal-code.
* **APS conformal sets (AIS)** ‚Äî calibration fold from dev; APS nonconformity = cumulative prob mass to include the true class; threshold at ‚åà(n+1)¬∑0.8‚åâ quantile ‚Üí `q_hat`. Mondrian variants under `aps_q_hat` + `aps_q_by_group`. Inference: sort class probs desc, accumulate until cumsum ‚â• resolved q_hat. **Coverage runs conservative (~99% vs 80% target) because K=5 makes APS scores discrete ‚Äî expected** (avg set ‚âà2.8).
* **LOS (log1p) head** ‚Äî same regression machinery; `y` log1p'd, conformal q in log-space; predictions/PI back-transformed via `expm1`, clipped `[0,‚àû)`. Metrics in raw days. LOS is hard in- and out-of-time (R¬≤‚âà0.2) and right-censored (¬ß1).
* **TreeSHAP** on the held-out **test set only** (training SHAP is optimistic), cached in `shap_test.joblib`; multiclass cache is 3-D `(n,p,K=5)`, AIS axis last (insight engine slices by `class_idx`). Also persists `shap_interaction` (regression `(n,p,p)`, multiclass `(n,p,p,K)`; diagonal=main effect, off-diag=symmetric pairwise; cats pre-encoded to int codes per ¬ß0b; `training_metrics.json` stores `global_interaction_top25`; dominant interaction = age √ó motor) and `y_test`/`y_pred`(/`y_pred_proba`) for the Methods calibration visuals.
* **Holm correction** ‚Äî running **max** over sorted p√ó(n‚àík+1), not min (¬ß0b).
* **Trajectory models** ‚Äî 9 independent LightGBM regressions of SCIM-total at 72h‚Ä¶6m from the same admission features. `models/trajectory/bundle.joblib` (`timepoints`,`models`,`conformal`,`clip_min/max`); each timepoint median/p10/p90 + Mondrian q. No SHAP. Admission features most predictive of ~1-month, least of ~5-month; conformal q widens monotonically. Value is trajectory *shape*, not per-timepoint accuracy.
* **Recovery archetypes** ‚Äî k-means (k=3 by silhouette) on predicted 10-point trajectories (9 timepoints + discharge), z-scored per timepoint. Ordered by discharge SCIM: 0 limited / 1 gradual / 2 rapid. `data/archetypes.py::assign_single()` assigns at runtime. `models/archetypes/archetypes.joblib`. Primary separators: AIS grade, age.
* **Temporal validation (`models/temporal.py`, F24)** ‚Äî out-of-time rolling-origin backtest, **diagnostic only**; writes `models/temporal_metrics.json`, never touches `train.py` artifacts. Expanding window, test years 2020‚Äì2025, train=`BusinessYear < T`, group-safe (test episodes whose patient appears in the past are dropped + counted `n_dropped_overlap`). Reuses train/conformal helpers; deltas: temporal cut, **marginal** conformal/APS (per-origin folds too small for Mondrian); reported coverage is conformal-only. Finding: SCIM lose only ŒîR¬≤‚âà‚àí0.04‚Ä¶‚àí0.08 OOT (coverage ‚âànominal); AIS robust (APS ~0.99); LOS hard both ways.

### Diagnostic/inference model families (G1‚ÄìG11) ‚Äî shared pattern
**State once, applies to G1/G2/G4/G6/G7/G8/G10/G11 unless noted:** grouped-5-fold-CV-by-`IDNumber` ‚Üí **out-of-fold (OOF)** predictions drive every metric, the Platt (sigmoid) calibrator for binary heads, and a **cross-conformal** APS q for multiclass (valid small-n analogue of production's single split-conformal); heads **refit on the full cohort** reusing the OOF calibrator/q (conservative). SHAP = **descriptive in-sample** drivers (not OOS). Binary plumbing (`_typed_X`/`_oof_binary`/`_fit_platt`/`_apply_platt`/`_refit`/`_calibration_curve`/`_shap_top`) is **imported verbatim from `conversion.py`**. Each module writes a **tracked identifier-free `models/<name>_metrics.json` + git-ignored `models/<name>/bundle.joblib`** and **never touches `train.py` artifacts (byte-repro verified)**. `_apply_platt` is **mirrored inline in `compute.py`** so the dashboard never imports the model module (it pulls shap via `train.py`/`conversion.py`). Bundle/metrics shapes are documented inline atop each module. `MIN_COHORT=120` (where applicable). Patient cards that are **admission-grade-gated** override `AIS_ord` with the episode's *raw* admission grade before inference (cohort membership is undefined when the grade is unrecorded, ~12%); **non-gated** cards (G7/G8/G11) use `episode_row_for_model`'s imputed row as-is, no override.

* **G1 landmark (`models/landmark.py`)** ‚Äî per landmark L‚àà{72h,2w,4w,6w,2m,3m} refit the **discharge** outcome on admission features **+ a LOCF observation block** (`LANDMARK_COLS`: SCIM total+3 subscales, AIS_ord, UEMS, LEMS, TotalMotor, LightTouch, PinPrick, each carried forward as last non-null at a timepoint ‚â§ L, prefixed `L_`; landmark feats=40 vs baseline 30). **Eligibility = still-admitted risk set** (episode enters L only if it has an observation at an intermediate timepoint with order-index ‚â• L; `discharge` excluded from `TIMEPOINT_ORDER`) ‚Äî avoids immortal-time/leakage. Each landmark paired with an **admission-only baseline refit on the identical eligible cohort+split** so the delta isolates the value of observation. Marginal conformal/APS. Persists **both heads per (outcome,L)** under `by_landmark[L]={"baseline","landmark"}` + `timepoint_order`. Finding: SCIM ŒîR¬≤ +0.04(72h)‚Üí+0.30(3m), PI half-width ‚âà23‚Üí13.
* **G2 value-of-information (`landmark.py` + `compute.landmark_voi`)** ‚Äî per (outcome,L) also fits 10 **single-add heads** (admission + exactly one `L_<measure>`, 31 feats), each its own marginal q; metrics under `by_landmark[L]["single"][measure]`, plus `lm_value_summary[L][measure]={median,q25,q75,n}` for cohort-median imputation. `landmark_voi(key,L,X_base,observed)` ranks measures for one patient: real value if present (`observed`, realized gain) else cohort median (`prescriptive`, hypothetical). **CRUX ‚Äî marginal q ‚áí a single-add head's PI half-width is a per-MEASURE constant, not per-patient**: for identity outcomes (SCIM/AIS) `d_halfwidth` is patient-independent (only the point estimate personalizes); **log1p LOS is the exception** (constant q in log-space, `expm1` scales it ‚áí patient-specific). So VOI bars rank a population quantity for SCIM/AIS, a personal one for LOS ‚Äî by design.
* **G3 phenotyping (`data/phenotypes.py` + `models/phenotypes.py`)** ‚Äî **multivariate growth mixture model** (mixture of linear mixed models) over *observed* early SCIM-total + total-motor trajectories jointly (contrast ¬ß3 archetypes = k-means on *predicted* curves). `y_{i,m}(t)=Œ≤_{k,m}¬∑œÜ(t)+b_{i,m}¬∑œà(t)+Œµ`; œÜ=poly degree D (BIC ‚àà{1,2}), œà=[1,t]. **Class-invariant `G` (block-diag) + `sigma2`** (avoids variance-collapse; `V_i=Z_i G Z_i'+R_i` class-independent, built once/individual/EM-iter; only mean is class-specific ‚Äî distinct from LCGA's G=0). **Native missingness** (each individual contributes only its observed cells) = the win over k-means-on-imputed. EM (log-sum-exp E; ECM M: resp-weighted GLS Œ≤ + Laird-Ware variance components), multi-restart, K√ódegree by BIC + min-class-share floor. Fit: **K=5, degree=2** (590 episodes ‚â•3 SCIM points). **CRUX ‚Äî degree-2 extrapolation traps:** means valid only over each class's *observed support* (an early-discharge class dives to SCIM ‚àí1277 at 6m). `class_support(...min_coverage=0.20)`‚Üílast window index where ‚â•20% still observed; (a) blank curves past support (+clip [0,100]); (b) `order_by_discharge` ranks by clipped SCIM at the **latest universally-supported** timepoint (raw 6m would mislabel the best early-discharge recoverer as worst). **Invariant: full-window membership reproduces the bundle's stored `posterior` row byte-for-byte for an in-cohort episode** (verified 0.0); generalizes to out-of-cohort episodes (fresh `predict_proba`). `compute.predict_phenotype_membership`, `phenotype_cutoff_options`.
* **G4 AIS conversion (`models/conversion.py`)** ‚Äî admission‚Üídischarge AIS *transition* (not absolute `y_discharge_ais`), admission grade kept as feature (conversion is admission-gated: ceiling at D, none at E). Heads: 2 **calibrated binary** (`motor_incomplete` A/B‚Üí‚â•C; `ambulatory` A‚ÄìC‚Üí‚â•D) + **ordinal magnitude** multiclass {0,+1,‚â•+2} (`MAG_CAP=2`) on the A‚ÄìD room-to-improve cohort (deterioration‚Üíclass 0). **CRUX ‚Äî binary vs magnitude are NOT comparable:** binary heads use no `class_weight` (near-balanced‚Üícalibrated); magnitude uses `class_weight="balanced"` (uncalibrated, inflated minority probs) ‚Äî they disagree (adm A: binary P(‚â•C)=0.51 vs magnitude P(‚â•+2)=0.76). UI: binary = the calibrated probability, magnitude = APS set / argmax (never a competing probability). **Requires `AIS_ord`** (gates: endpoint applies only if adm grade ‚àà its `adm_grades`; magnitude A‚ÄìD only; `ais_ord=None`‚Üí"needs-grade" prompt). Findings: `ambulatory` AUC‚âà0.87; `motor_incomplete` AUC‚âà0.62 (admission poorly predicts complete‚Üíincomplete ‚Äî a real clinical reality); magnitude Œ∫‚âà0.49.
* **G6 AIS multi-state (`models/multistate.py`)** ‚Äî AIS-grade *trajectory* over the dense `0day..6m` grid. (1) **Population time-inhomogeneous Markov** over 5 states (A=1..E=5): per step the empirical `P_k` from **pairwise-complete** episodes, zero-departure row ‚Üí identity (mass-conserving); forward-multiply from an admission point mass, **stratified by admission grade**, for occupancy, first-passage conversion-to-‚â•X (‚â•X made absorbing ‚áí monotone; labels `improve`=‚â•adm+1, `ge_C`, `ge_D`), median day to improvement, expected days/state (`np.trapezoid`, NOT numpy-1 `np.trapz`). **CRUX ‚Äî apparent regressions are real** (B‚ÜíA, D‚ÜíC reassessment/inter-rater noise ‚Üí backward mass; surface honestly; small-n cells expose it). (2) **Covariate improve-by-6m binary** (P(‚â•1-grade improvement) on A‚ÄìD cohort with ‚â•`MIN_WINDOW_OBS=2` in-window obs; no class_weight‚Üícalibrated). AUC‚âà0.90, Brier‚âà0.12. **Clinical finding ‚Äî improve base rate is NON-monotone in admission grade: B highest ~85%, C ~74%, A ~64%, D lowest ~15%** (D-improvement means reaching **E**=full-normal; UI frames D's head as P(D‚ÜíE), `to_e=True`). **Cohort, not personalized** ‚Äî occupancy/conversion/sojourn depend only on admission grade; only the `improve` prob + own-trajectory overlay differ between same-grade patients. First-passage dedup: `improve`(=adm+1) coincides with ‚â•C/‚â•D for B/C admissions ‚Äî map to distinct target grades to avoid duplicate curves. E admission = `at_ceiling` (occupancy only).
* **G7 functional-independence (`models/independence.py`)** ‚Äî **18 calibrated binary heads**, P(**functional independence** at discharge: aids/devices allowed, no human assistance) per SCIM-III ADL item (Respiration excluded, 95% independent ‚Üí too imbalanced). **Threshold table (`independence.py::ITEMS`, load-bearing):** feeding/bathing/dressing/grooming **‚â•2**; bladder **‚â•9** / bowel **‚â•8**; toilet **‚â•3**; bed-mobility =6 and the four transfers (bed‚ÜîWC, WC‚Üîtoilet, WC‚Üîcar, ground‚ÜîWC) at **top score** (scale lumps human-assist with device-use); the 3 **walking** items (indoors/moderate/outdoors) **‚â•4** = ambulates with-or-without aids (wheelchair-independence excluded ‚Äî distinct ambulation milestones). Domains: self_care(6)/sphincter(3)/mobility(5)/ambulation(4). **CRUX ‚Äî uncertainty is the calibrated prob + reliability curve, NOT a conformal set (binary APS degenerates, ¬ß0b ‚Äî removed).** Findings: mean AUC‚âà0.905 (range 0.88‚Äì0.93); **expected-independent count** (Œ£ of 18 probs) rises monotonically with admission AIS (A‚âà0.9‚ÜíB‚âà6‚ÜíC‚âà13‚ÜíD/E‚âà17 of 18); outdoor walking hardest (P‚âà0.36 even at AIS-C). **NOT grade-gated** (no real-grade override). **CRUX ‚Äî `AIS_ord` alone is a weak driver (TotalMotor/UEMS dominate SHAP): a Simulator profile with only an admission grade is flat (~8.5/18), NOT grade-differentiated; the A‚ÜíE spread emerges only when correlated motor/sensory move with the grade** (honest under missingness, consistent with F25).
* **G8 topography (`models/topography.py`)** ‚Äî per-ISNCSCI-segment recovery atlas: for **all 132 segments** (20 motor + 56 LT + 56 PP, L/R), calibrated P(milestone at discharge): motor **P(grade‚â•3)**=antigravity, sensory **P(grade‚â•1)**=preserved sensation. **Method = 132 INDEPENDENT calibrated binary heads (NOT the low-rank model first chosen). CRUX ‚Äî a diagnostic settled it: the 30 aggregate features predict a *specific* segment near-chance (OOF AUC 0.43‚Äì0.63); the dominant predictor is the segment's OWN admission grade (diagonal, not cross-segment), and the discharge matrix isn't low-rank. Adding `adm_self` (the segment's own admission grade, LOCF, still admission-only) lifts AUC to ~0.93.** **Alignment invariant: align per-segment matrices to `ep` by `ep["KeyRecordNumber"]`, NOT `ep.index` (¬ß0b) ‚Äî co-misalignment keeps AUC high while scrambling context+groups; verify with the behavioral check.** 12 degenerate heads (high-cervical C2/C3 LT+PP, < `MIN_MINORITY=12`) carry constant base-rate, no model. Findings: mean OOF AUC motor 0.94/LT 0.93/PP 0.91. **The expected-antigravity count is driven by injury LEVEL, not AIS ‚Äî it is NON-monotone in admission AIS (A 12.7, B 16.3, C 14.5, D 15.2, E 15.8) ‚Äî a "rises with AIS" sanity check is WRONG here** (unlike SCIM/independence). **NOT grade-gated.**
* **G9 Œî score-recovery (G9 ‚Äî `outcomes.py` registry, NOT a standalone module)** ‚Äî predicts admission‚Üídischarge *change* in 5 ISNCSCI summary scores: ŒîUEMS, ŒîLEMS, Œîtotal-motor, Œîlight-touch, Œîpin-prick (`delta_{uems,lems,totalmotor,lighttouch,pinprick}`), the canonical SCI-trial endpoint. **Pure production-registry extension** (two edits: 5 `OutcomeSpec`s into `OUTCOMES`; 5 `y_delta_*` targets in `build_episode_frame` = discharge-slot ‚àí the same first-non-null admission feature; NaN either side‚ÜíNaN, dropped). `train.py` untouched; reuses SCIM regression machinery (median/p10/p90 + Mondrian conformal + SHAP, identity transform). **No leakage** (label is the delta). **Negative-range invariant: `clip_min < 0`** (¬±50 motor / ¬±100 total-motor / ¬±112 sensory) so deterioration is representable; `clip_min < 0` is also the UI's unique **Œî-head flag** ‚Üí signed point/PI (`+11`/`‚àí3`), drops the "/ max" suffix, draws a dotted "no change" zero-line spanning ¬±range. **Behavioral gate** (not R¬≤): at-ceiling patient predicts Œî‚âà0, high-headroom large +Œî (ŒîUEMS +1.2 vs +29.1). Finding: motor recovery more predictable than sensory; dominant SHAP shifts to `mFrankel_ord √ó <matching baseline score>`. F24 temporal and G1/G2 landmark **iterate all `OUTCOMES`** (`for spec in OUTCOMES`; the only skip is `if not info: continue` for an empty cohort cell ‚Äî NOT a Œî exclusion), so they DO cover the 5 Œî heads, and well (delta_uems landmark R¬≤ 0.60‚Üí0.82 across 72h‚Üí3m). Their committed JSONs had been **stale (pre-G9, 6 outcomes)** until F27's retrain regenerated both to all 11 ‚Äî a real latent inconsistency the dep refresh surfaced and fixed.
* **G10 level descent (`models/level_descent.py`)** ‚Äî admission‚Üídischarge change in 5 ISNCSCI neurological *levels* (NLI + bilateral motor + bilateral sensory), complement to G9's Œî scores. Cord-level ordinal **C1=0..S45=28** (smaller=more rostral=more severe); **Œî>0 = caudal descent = improvement**. **`INT‚ÜíINT_ORD=29` ceiling:** full recovery (raw `INT`, which the loader maps to NaN) lifted to ord 29 at discharge so a cure is the Œî ceiling, not a dropped row; admission baseline uses loader `*_ord` (INT/missing‚ÜíNaN) so already-intact admissions drop (only ord-0..28 enter). **2 heads/level (10):** calibrated binary P(Œî‚â•1) (no class_weight, Platt) + ordinal magnitude {0,+1,‚â•+2} (balanced, APS) ‚Äî same binary-vs-magnitude CRUX as G4. Findings: **only NLI well-predicted (AUC‚âà0.73); bilateral motor/sensory AUC‚âà0.62‚Äì0.63 ‚Äî a genuine admission-signal ceiling** (single-boundary threshold-crossing is noisy). **Report the median** (INT‚Üí29 inflates NLI mean +3.1 vs median +1). **Enrichment rejected**: G8-style concat of per-segment admission grades left AUC flat-to-worse (¬ß0b state-vs-threshold-crossing lesson).
* **G11 neuro-functional dissociation (`models/dissociation.py`)** ‚Äî predicts the *mismatch* between neurological (Œî ISNCSCI score) and functional (Œî SCIM subscale) recovery. **Score `D = z(Œîfunction) ‚àí z(Œîneuro)`** (D>0 = functional over-achiever / compensation; D<0 = neurology outpaces function). **Contrast, not a regression residual** ‚Äî the axes are weakly coupled (Pearson r‚âà0.21/0.12/0.27) so a residual would collapse to the functional axis (¬ß0b). 3 domain-paired axes: `uems_selfcare`, `lems_mobility` (the r‚âà0.12 decoupling: lower-limb strength barely predicts mobility independence ‚Äî wheelchair skills), `totalmotor_total`. **2 heads/axis: over_achiever** = calibrated binary P(D>0); **magnitude** = continuous regression on D + marginal cross-conformal 80% PI. **Inverse of the G4/G10 CRUX ‚Äî magnitude is a regression (no class weighting) so the two heads ARE honestly comparable** (sign(magnitude)‚âàbinary direction); surface binary=prob, magnitude=signed point+PI. All axes n=438. Findings: binary AUC 0.93‚Äì0.95, magnitude R¬≤ 0.69‚Äì0.76; dominant driver = the NEURO-axis baseline score (ceiling/room), age #2. **Honest-novelty caveat (¬ß0b): predicted-D ‚âà0.97-correlates with naively differencing two separately-modeled axes ‚Äî value is the framing (one calibrated signed dissociation + PI), not a new mechanism. Star-placement CRUX (UI): a 1-D D has no 2-D position without an external anchor ‚Äî x = the patient's predicted Œîneuro from the matching G9 head, vertical gap = sd_f¬∑D.** **NOT grade-gated.**

## 4. Dashboard conventions

* **Module layout** (`src/rehab_sci/dashboard/`) ‚Äî browse the per-file symbol inventory via Serena LSP (`get_symbols_overview`); only the non-obvious contracts live here:
  - `state.py` ‚Äî all startup globals, loaded once; depends only on theme + data/model layers (no other dashboard module imports it transitively).
  - `compute.py` ‚Äî pure (model inference, conformal-q, APS, SHAP, row-prep), **no Dash/Plotly**. Landmark inference lives here: `predict_landmark` (paired baseline+landmark heads from `LANDMARK_BUNDLE`), `landmark_observed_for_episode` (real LOCF block on the still-admitted risk set), `episode_landmark_eligibility` (per-L still-admitted gate).
  - `reliability.py` ‚Äî pure; `assess_input(X, bundle, feature_spec)` ‚Üí importance-weighted completeness + OOD (range violations / atypicality vs `feature_spec['ranges']` q05‚Äìq95) for the simulator partial-input badge.
  - `layout.py` ‚Äî the shared *prediction* figures (`fig_shap_local`, `fig_prediction_interval`, `fig_class_probabilities`) + landmark compare (`fig_landmark_compare` + `landmark_readout`), shared by simulator & patient cards ‚Äî **not** in the figures package.
  - `figures/` ‚Äî a **package** split by tab (`overview`,`insights`,`patient`,`methods`,`simulator`) + `_common`; every public name re-exported via `figures/__init__.py` `__all__` (so `from rehab_sci.dashboard import figures as fg` + every `fg.fig_*` are unchanged). Add a figure: put it in its tab submodule and append the name to `__all__`.
  - `tabs/*` = per-tab layout + `@callback`s; `app.py` = entry + chrome callbacks.
* **Dependency graph (acyclic):** `state`‚Üídata/model; `compute`‚Üí`state`; `layout`‚Üí`state`,`compute`; `tabs/*`‚Üí`state`,`compute`,`layout`,`figures`; `app`‚Üí`tabs/*`. Importing a tab triggers `@callback` registration (Dash function-level `@callback` registers globally ‚Äî no `@app.callback`).
* `dcc.Store("lang-store")` holds `"ja"`/`"en"`; most text callbacks take it as `Input` for instant swaps. **Exception:** `update_overview_content` takes it as `State` to avoid a race with `update_tab` (both fire on lang change ‚Äî `State` makes overview fire *after* `update_tab` rebuilds the filter components with new-lang labels).
* Pattern-matched simulator inputs use IDs `{"type":"num"/"cat","col":<raw>}` with `dash.ALL`; order fixed by `feature_spec['feature_cols']`. Numeric fields are **clearable `dcc.Input(number)`** (not sliders); the form opens **blank** (F25) ‚Äî `collect_sim_inputs` leaves blanks as NaN (no imputation); *Fill cohort defaults* / *Clear all* set every value via one pattern-matching value-Output callback; What-if mode prefills from the reference episode. (`simulate` returns 5 outputs incl. `sim-reliability`.)
* Plotly template `"medical"` (registered in `theme.py`). Palettes `PALETTE_CATEGORICAL`, `PALETTE_AIS` (A‚ÜíE cool‚Üíwarm), `PALETTE_PARA`, `PALETTE_ARCHETYPE` ‚Äî use them for every chart. Japanese rendering needs the font stack `"Hiragino Sans","Noto Sans JP","Yu Gothic UI"` in both Plotly and CSS.
* `dcc.Store("patient-ref")` (session-scoped) carries the What-if reference `{id_number,key_record,features,outcomes,trajectory}`; `update_tab` reads it as `State` to prefill simulator defaults, `simulate` reads it as `State` for the reference overlay.

## 5. Known gotchas

* **`IntCastingNaNError` on `IDNumber`** ‚Äî patients with no admission row produce NaN IDs; `dropna(subset=[outcome,"IDNumber"])` then cast `float64‚Üíint64`.
* **Stale dashboard process** ‚Äî `kill <PID>` stops only the `uv run` wrapper; the Python child keeps serving old code. `pkill -f 'rehab_sci.dashboard.app'` **self-matches the calling shell** (the pattern is in its own argv) ‚Üí the shell is signalled too (the exit-144 / signal-16 report). So: (1) it still kills the dashboard, fine as the **last** statement of a cleanup-only command; (2) **put it after anything else you still need to run**, or the shell dies at signal 16 first (empty logs, nothing listening ‚Äî looks like a boot failure but is not). Prefer: launch via the Bash tool's background mode (no pkill), poll readiness in a **separate** command containing no matching literal (curl only), stop by port `fuser -k 8050/tcp` (never self-matches). Verified boot = `curl -s -o /dev/null -w '%{http_code}' http://127.0.0.1:8050/` ‚Üí 200, plus `/_dash-dependencies` for the callback graph.
* **pandas fragmentation warning** when adding many columns serially ‚Äî batch via `pd.concat([df, new_cols_df], axis=1)` (loader already does this).
* **`@dataclass(frozen=True)` + dict fields** breaks under `@lru_cache` on instance methods (unhashable) ‚Äî `Schema` stays a plain `__slots__` class.
* **`kaleido<1`** has no Linux x86_64 wheel under the current resolver ‚Äî keep `kaleido>=1.0,<2`. Plotly‚ÜíPNG needs Chrome (`kaleido.get_chrome_sync()` at first use).
* **Packaging** ‚Äî a real hatchling uv package (`[tool.hatch.build.targets.wheel] packages=["src/rehab_sci"]`); `uv sync` installs it editable, so `uv run python -m rehab_sci.*` needs no `PYTHONPATH`.
* **Background dashboard from a bash one-liner** ‚Äî `nohup ‚Ä¶ &` inside the harness wrapper may not survive wrapper exit (exit 144 = SIGTERM bookkeeping); use the Bash tool's `run_in_background: true`, or make the launch the *last* statement.
* **`Sunburst(branchvalues="total")` requires parent value = sum of children** ‚Äî parent `0` renders blank silently. Prefer accumulating leaf counts into every ancestor (hover shows true subtotals) over `branchvalues="remainder"`. (The injury chart later moved to a treemap.)
